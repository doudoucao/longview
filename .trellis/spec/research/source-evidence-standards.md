# Source and Evidence Standards

## Source Hierarchy

Prefer sources in this order when available:

1. Primary sources: official documentation, public filings, regulator publications, standards bodies, original papers, official repositories, and direct product materials.
2. High-quality secondary sources: reputable financial media, analyst reports available to the user, academic surveys, and industry reports with transparent methodology.
3. Community or informal sources: blogs, forums, social posts, and newsletters. Use these for leads or practitioner color, not as sole support for important claims.

## Citation Requirements

Every material claim in a final research artifact must include either:

- a source link or local file path
- a clear statement that the claim is an inference based on listed sources
- a note that the claim is an assumption requiring validation

Do not cite a source that only weakly supports the sentence. If the source supports only part of the claim, narrow the claim.

## Freshness Rules

Use current verification for:

- AI model, agent framework, or product capabilities
- pricing, packaging, API availability, and licensing
- financial regulations and exchange rules
- market data provider coverage and terms
- statements about adoption, rankings, or "leading" status

When a source is older but still useful, label it as historical context.

## Confidence Labels

Use:

- High: primary source or multiple independent reliable sources agree.
- Medium: credible secondary source or primary source with partial ambiguity.
- Low: single weak source, marketing-only claim, or inference from incomplete evidence.

Low-confidence claims should not drive final recommendations without review.

## Contradictions

If sources conflict, preserve the contradiction. Explain what differs, which source is more authoritative, and what would resolve the gap.
