# 调研 Agent 自动化投资研究范式与工具

## 目标

调研全球先进的 agent 自动化投资与研究范式、工具、skills、数据获取方式、工作流和自动化实现模式，为 Longview 这个深度价值投资社区 / 结构化投研平台提供方向判断。

本任务的产出应帮助 Longview 定义：如何让 AI agent 参与投资研究，如何获取和组织数据，如何沉淀工具/skills，如何编排自动化工作流，以及如何保存中间原始材料和可追溯证据链，同时避免把自动化输出包装成个性化投资建议。

## 已确认事实

- `README.md` 中的仓库定位是：Longview 是“深度价值投资社区 / 结构化投研平台方案仓库”。
- 当前仓库只有 Trellis 初始化结构，还没有应用代码；因此本任务应先建立研究规范和规划产物，再进入后续实现。
- 本任务偏重调研，且信息时效性强：AI agent 模式、投研工具、市场数据 API、监管要求和公开案例都可能快速变化。
- 金融研究属于高风险信息场景，产出必须区分事实、估计、观点和模型推断。
- 用户明确希望 agent 帮助调研当前世界上先进的 agent 自动化投资与研究范式和工具/skills，调研来源包括论文、博客、GitHub、官方文档、公开产品资料等，并要求保存中间原始内容。

## 需求

- 梳理先进的 agent 自动化投研范式：
  - agentic research 工作流
  - autonomous analyst / copilot / reviewer 分工
  - RAG 与投研知识库
  - 基于 SEC / 财报 / 公告的基本面分析管线
  - 市场、新闻、事件监控系统
  - 多 agent 辩论、reviewer 和 red-team 流程
  - human-in-the-loop 研究审核机制
  - 结合组合与风险约束的研究编排
- 调研 agent 自动化投研所需的数据获取方式：
  - 监管披露和财报数据，例如 SEC/EDGAR、交易所公告、公司 IR
  - 市场行情和基本面数据 API
  - 新闻、博客、公告、社媒、事件流和宏观数据
  - GitHub、论文、技术博客、产品文档等技术来源
  - 数据许可、访问限制、更新频率、结构化程度和可自动化程度
- 对比代表性工具、skills、框架和参考实现，包括开源 agent 框架、研究编排模式、数据抽取管线、金融数据工具、浏览器/爬取工具、filing 工具、代码执行工具，以及公开资料可验证的投研产品。
- 总结可复用的 agent skills/tooling 清单：
  - 搜索与来源发现
  - 网页/PDF/公告/财报解析
  - 结构化抽取与 schema 校验
  - 估值、财务建模和同业比较辅助
  - 新闻/事件监控与提醒
  - 证据引用、事实核查和反驳
  - 报告生成、审阅和版本化
- 梳理可自动化的工作流模式：
  - 从研究问题到来源发现
  - 从原始材料保存到结构化笔记
  - 从证据库到假设生成
  - 从初稿到 reviewer / red-team / human approval
  - 从一次性调研到持续监控和更新
- 定义 Longview 对 AI 投研内容应采用的证据标准：
  - 来源层级
  - 引用要求
  - 声明验证
  - 时效性检查
  - 不确定性标注
  - 审计轨迹保留
- 保存调研中间原始内容：
  - 论文、博客、GitHub README/代码片段、官方文档、产品页、报告和其他重要来源必须保留原始链接与本地摘要/摘录记录
  - 对关键来源建立结构化 source note，记录 URL、访问日期、来源类型、关键内容、限制、相关性和置信度
  - 对无法完整保存的内容记录原因，例如版权限制、登录限制、付费墙或 robots/访问限制
- 定义适合 Longview 的可执行研究运营模型：
  - 研究对象模型
  - 来源摄取策略
  - analyst / agent 角色分工
  - 质量门禁
  - 合规和免责声明边界
  - MVP 工作流候选方案
- 输出可执行建议，但本任务不产出任何具体证券的买入、卖出或持有建议。

## 约束

- 必须使用当前来源。由于主题变化快，在描述“最新”“领先”“state-of-the-art”方案前，必须联网或通过其他方式验证最新信息。
- 优先使用一手来源：论文、官方文档、公开披露文件、监管材料、公司官方材料、原始代码仓库、release notes、API 文档和项目 README。
- 可以使用高质量博客、工程实践文章、行业报告和媒体报道作为补充，但不能把营销表述当作已验证事实。
- 必须清楚标注来源类型和置信度，并区分直接证据与推断。
- 必须保存中间原始内容或可复核的原始材料索引；不能只保留最终综合结论。
- 避免把研究内容写成个性化投资建议。
- 必须解释数据缺口、覆盖范围限制和关键假设，以维持用户信任。

## 验收标准

- [ ] `research.md` 或等价最终报告解释先进的 agent 自动化投资研究范式，并说明各自适用场景、优势和弱点。
- [ ] 报告至少对比 12 个相关工具、skills、框架、产品或公开参考实现，并提供引用来源。
- [ ] 报告单独覆盖“如何拿数据”：数据源类型、获取方式、接口/抓取/下载路径、许可限制、更新频率、结构化难度和自动化适配度。
- [ ] 报告单独覆盖“如何自动化”：agent 角色、工具调用、数据流、状态管理、review gates、human-in-the-loop 和持续监控方式。
- [ ] 报告包含可复用的 agent skills/tooling 清单，并标注每个 skill 的输入、输出、依赖数据、失败模式和适用场景。
- [ ] 报告包含适合金融研究场景的来源质量评分标准和声明验证流程。
- [ ] 报告提出 Longview 专属的自动化投研目标运营模型。
- [ ] 报告包含 MVP 路线图，明确第一批实验和非目标。
- [ ] 报告明确覆盖合规和安全边界，包括非投资建议表述和人工审核机制。
- [ ] 所有关于当前工具、产品、法律、市场结构或模型能力的重要声明都有来源支持，或明确标记为推断。
- [ ] 中间原始材料被保存到任务目录下的 `sources/` 或等价目录，并有 source index 说明每份材料的来源、访问日期、保存形式、用途和限制。

## 非目标

- 构建完整研究平台。
- 为任何具体证券产出投资建议。
- 回测交易策略。
- 未经授权抓取付费墙或访问受控数据。
- 大规模保存受版权保护的全文内容；必要时只保存短摘录、元数据、链接和研究笔记。
- 在没有当前权威来源支持的情况下，对监管合规作结论性判断。
